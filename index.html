Web 퀴즈!

  1. HTTP Method 종류를 아는대로 설명해주세요.

    GET: 리소스를 조회합니다.
    POST:데이터를 추가,등록합니다.
    PUT:리소스를 대체,수정하고 해당 리소스가 없으면 새롭게 생성합니다.
    DELETE:리소스를 삭제합니다.
    PATCH:리소스 부분을 변경(수정)합니다.
    cf)PUT이 전체를 변경하고 PATCH는 일부를 변경합니다.
    HTML 태그를 적극 활용해도 좋습니다.

  2. 인터넷과 웹의 차이에 대해 설명해주세요.

    웹은 인터넷을 구성하는 기술들 중 하나의 기술이고, 인터넷에 연결된 사용자들이 서로의 정보를 공유할 수 있는 공간을 의미합니다.
    웹페이지는 하나의 문서인데 이러한 문서들을 연결하는 것이 웹입니다.
    현재 웹 기술에서는 요청과 응답을 HTTP를 통해 통신합니다.
    인터넷이란 여러 통신망을 하나로 연결한다는 의미의 ‘inter-network’라는 말에서 시작되었으며,
    이제는 전 세계 컴퓨터들을 하나로 연결하는 거대한 컴퓨터 통신망을 의미합니다.
    인터넷은 클라이언트와 서버로 구성되며, TCP/IP라는 기본 프로토콜을 통해 제공되고 있습니다.

  3. www.naver.com을 치면 무슨 일이 일어날까요?

    1.브라우저의 URL 해석/파싱
    cf)파싱이란? 컴퓨터 프로그래밍에서 특정 형식으로 구성된 데이터를 분석하고 그 의미를 이해하는 과정
    https://www.naver.com https는 프로토콜입니다. www.naver.com은 도메인 이름입니다.
    어떤 프로토콜을 사용할 것인지, 어떤 URL로 서버에 요청을 보낼 것인지 해석하는 과정이 이루어집니다.

    2.DNS서버에서 IP주소 받아오기 (URL -> IP Address)
    인간이 이해하기 쉽도록 도메인 이름을 사용하던 URL을 컴퓨터가 이해할 수 있도록 IP주소로 변경해야 합니다.
    이러한 역할을 DNS서버가 합니다.

    3.라우터를 이용해 접속하려는 네트워크로 가는 최적의 경로 찾기

    4.DNS서버에서 응답 받은 IP주소를 MAC주소로 변환
    cf)IP와MAC의 차이점 : 우리가 접속하려는 서버의 네트워크를 찾기 위해 IP주소를 사용하고, 그 네트워크 내부에 있는 컴퓨터와 통신하기 위해 MAC 주소가 필요한 것입니다.

    5.TCP 소켓 통신
    cf)소켓이란 : 소프트웨어로 작성된 추상적인 개념의 통신 접속점입니다. 소켓을 통하여 통신망으로 데이터를 송수신하게 됩니다.

    6.HTTPS(HTTP) 프로토콜 요청/응답
    naver.com 페이지를 달라고 서버에 요청을 보냅니다.
    서버는 응답을 받고, 응답이 가능하다면 브라우저에게 해당 요청에 대한 응답을 생성(HTML,CSS,JS)하여 전달합니다.

    7.브라우저가 응답
    응답(HTML,CSS,JS등)을 받은 브라우저는 웹 페이지를 표시합니다.
